# prd-creator

대규모 기능의 PRD(Product Requirements Document)를 작성합니다.  
에픽 단위의 큰 기능을 `plan-creator`로 처리할 수 있는 독립적인 하위 태스크로 분해합니다.

## 언제 사용하나요?

- 여러 개발 세션에 걸쳐 구현해야 할 큰 기능을 기획할 때
- 전체 범위와 방향을 잡고 개별 태스크로 분해하고 싶을 때
- 팀과 공유할 구조화된 기능 명세가 필요할 때

## 트리거 문구

```
"PRD 작성해줘"
"전체 계획 세워줘"
"기능 전체 계획"
"에픽 계획"
"prd 만들어줘"
"작업 목록 정리해줘"
```

## 실행 흐름

1. **코드베이스 탐색** — 영향 받는 도메인, 서비스, 컨트롤러 파악
2. **정보 수집** — 배경/목표, 도메인 컨텍스트, 비즈니스 불변성, 범위, 하위 태스크 분해 방식에 대해 4~6개 질문
3. **PRD 문서 작성** — `prd-{feature}.md` 생성 (프로젝트 루트)
4. **피드백 요청** — 하위 태스크 분해 방식, 범위, 우선순위 검토

## 생성 문서 구조

```
prd-{feature}.md
├── 배경 및 목표
├── 도메인 컨텍스트 & 비즈니스 불변성
├── 범위 (in-scope / out-of-scope)
├── 하위 태스크 목록 (T1, T2, ...)
├── 태스크 진행 현황 테이블
└── 성공 기준
```

## 다음 단계

PRD 완성 후 각 태스크를 시작할 때:

```
"[T1 작업명] 계획 작성해줘" → plan-creator 실행
```

## 워크플로우 위치

```
prd-creator (전체 기획)
└── plan-creator (태스크별 구현 계획)
    └── tdd-team / test-driven-development (구현)
```
